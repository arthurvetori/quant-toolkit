<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Excel cell â†’ UDF | Untrusted input (could be malicious formula, overflow, injection) |
| UDF â†’ Handle Store | Concurrent access to shared store (data race, RefCount corruption) |
| UDF â†’ Native QuantLib.dll | P/Invoke boundary (DLL not found, architecture mismatch, segfault risk) |
| UDF â†’ File system | Logging writes could fail or be denied; exception handling must fail-soft |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Handle Store (RefCount) | Mitigate | Use Interlocked.Increment/Decrement for atomic updates; test concurrency with stress test (Task 6) |
| T-01-02 | Denial of Service | Handle Store (memory exhaustion) | Accept | No per-user rate limiting in v1; assume trusted Excel environment; document cleanup procedures |
| T-01-03 | Repudiation | UDF error handling | Mitigate | All errors logged with timestamp, function name, and context; audit trail in log file |
| T-01-04 | Information Disclosure | Log file (contains calculations) | Accept | Logs stored locally; user responsible for securing files; no PII in logs by design |
| T-01-05 | Information Disclosure | Handle IDs | Accept | Handle IDs are GUIDs, opaque to users; no sensitive data encoded in string |
| T-01-06 | Elevation of Privilege | Excel UDF execution context | Accept | Excel already has host-level privileges; UDF runs with Excel's permissions |
| T-01-07 | Denial of Service | Native DLL loading failure | Mitigate | Graceful error: SafeExecute returns error message if QuantLib.dll not found; clear diagnostic logging |
| T-01-SC | Tampering | NuGet package integrity | Mitigate | Use package signing verification during restore; `dotnet restore --locked-mode` if lock file created |

## Mitigation Details

**T-01-01 (RefCount corruption):**
- Code: `Interlocked.Increment(ref entry.RefCount)` â€” atomic operation guaranteed by CLR
- Validation: Concurrency stress test creates 1000 handles concurrently, verifies RefCount consistency post-cleanup
- Owner: Task 6 (concurrency test)

**T-01-07 (DLL loading failure):**
- Mitigation: SafeExecute catches DllNotFoundException, returns user message: "QuantLib library not loaded. Contact support."
- Logging: LogException includes DLL search paths attempted
- Owner: Task 3 (error handling layer)

**T-01-SC (package integrity):**
- Check: `dotnet restore --verify-match` during CI (future phase)
- For Phase 1: Document in setup guide to verify package hashes manually from nuget.org
