Build the centralized UDF registration framework that Wave 2-3 functions will inherit from:

**UdfRegistry.cs — Static registration entry point:**
- Static class with static constructor; ExcelDNA will invoke this at load time
- Method: `public static void RegisterUdfs()` — called by ExcelDNA on add-in load
- Purpose: Enumerate all UDF classes and log that they're available
- Simple for Phase 1 (ExcelDNA auto-discovers [ExcelFunction] attributes; this is just for diagnostics)
- Log: "QuantLib Excel Add-in v1.0 loaded; X UDFs registered"

**UdfBase.cs — Reusable base class for all UDFs:**
- Abstract or sealed utility class; not meant for inheritance in Phase 1, but provides shared logic
- Static method: `protected static object ExecuteUdf&lt;T&gt;(Func&lt;T&gt; fn, string functionName) → object`
  * Wrapper around SafeExecute from Core layer
  * Logs UDF call details: function name, timestamp, thread ID
  * Catches and formats exceptions to Excel safely
  * Per D-COMPOSE-03 (thread-safe), all UDF calls go through this bottleneck
- Static method: `protected static string CreateHandleFor(object obj, string description) → string`
  * Convenience method: calls HandleStore.Instance.CreateHandle, logs handle creation
  * Per D-COMPOSE-01 (opaque handles), returns the GUID string
- Static method: `protected static object GetHandleValue(string handleId, string expectedType) → object`
  * Convenience: calls HandleStore.Instance.GetHandle
  * Per D-COMPOSE-02, retrieves object by opaque handle
  * Validates type (fail with descriptive error if type mismatch)
  * Logs handle retrieval for diagnostics

**Key design principles for Phase 1:**
- All UDFs inherit these patterns; this ensures consistency
- Error handling is standardized (no UDF can bypass SafeExecute)
- Handle lifecycle is visible in logs (creation, retrieval, release)
- Minimal dependencies; no complex abstractions yet

**2. QL_BuildDICurve(double[] rates, double[] tenors) — Builder returning handle**
- Category: "QuantLib"
- Parameters: rates (array), tenors (array) — manual market data entry per MKTDATA-01
- Returns: string (opaque handle to cached curve object)
- File: Builders/CurveBuilder.cs
- Per D-COMPOSE-01: builder returns opaque string handle
- Implementation stub:
```csharp
[ExcelFunction(Name = "QL_BuildDICurve", Category = "QuantLib", Description = "Build DI/CDI curve from rates and tenors")]
public static object BuildDICurve(
    [ExcelArgument(Description = "Discount rates")] double[] rates,
    [ExcelArgument(Description = "Time to maturity (years)")] double[] tenors)
{
    return ExecuteUdf(() => {
        // Validation
        if (rates == null || tenors == null || rates.Length != tenors.Length)
            throw new ArgumentException("Rates and tenors must have equal length and not be null");
        if (rates.Length &lt; 2)
            throw new ArgumentException("At least 2 points required to build curve");
        
        // For Phase 1: stub implementation (no real QuantLib call yet; Phase 2 will add actual curve construction)
        var curveName = $"DICurve_{rates.Length}pts_{DateTime.Now.Ticks}";
        
        // Create a placeholder object (Phase 2 will replace with real QuantLib curve)
        var curveObject = new { Name = curveName, RateCount = rates.Length, TenorCount = tenors.Length };
        
        // Per D-COMPOSE-01: return opaque handle string
        return CreateHandleFor(curveObject, curveName);
    }, "QL_BuildDICurve");
}
```

**3. QL_GetCurveRate(string curveHandle, double timeInYears) — Consumer accepting handle**

```csharp
[ExcelFunction(Name = "QL_GetCurveRate", Category = "QuantLib", Description = "Query rate from curve handle")]
public static object GetCurveRate(
    [ExcelArgument(Description = "Curve handle from QL_BuildDICurve")] string curveHandle,
    [ExcelArgument(Description = "Time to maturity (years)")] double timeInYears)
{
    return ExecuteUdf(() => {
        if (string.IsNullOrEmpty(curveHandle))
            throw new ArgumentException("Curve handle is required");
        if (timeInYears &lt; 0)
            throw new ArgumentException("Time must be non-negative");
        
        // Per D-COMPOSE-02: retrieve object by opaque handle
        var curveObject = GetHandleValue(curveHandle, "curve");
        
        // For Phase 1: stub (Phase 2 will call real QuantLib methods)
        // Return a dummy discount factor: e^(-rate * T) ≈ 1 - 0.05 * T (5% flat rate)
        return Math.Exp(-0.05 * timeInYears);
    }, "QL_GetCurveRate");
}
```

**4. QL_BuildVolSurface(double[] strikes, double[] tenors, double[][] vols) — Builder returning handle**
- Category: "QuantLib"
- Parameters: strikes (array), tenors (array), vols (2D array of volatilities)
- Returns: string (opaque handle)

- Per D-COMPOSE-01: builder pattern

```csharp
[ExcelFunction(Name = "QL_BuildVolSurface", Category = "QuantLib", Description = "Build volatility surface from strikes, tenors, and vols")]
public static object BuildVolSurface(
    [ExcelArgument(Description = "Strike prices")] double[] strikes,
    [ExcelArgument(Description = "Time to expiry (years)")] double[] tenors,
    [ExcelArgument(Description = "Volatility matrix (strikes × tenors)")] double[,] vols)
{
    return ExecuteUdf(() => {
        if (strikes == null || tenors == null || vols == null)
            throw new ArgumentException("All inputs required");
        if (vols.GetLength(0) != strikes.Length || vols.GetLength(1) != tenors.Length)
            throw new ArgumentException("Vol matrix dimensions must match strikes and tenors");
        
        var surfaceName = $"VolSurface_{strikes.Length}x{tenors.Length}_{DateTime.Now.Ticks}";
        var surfaceObject = new { Name = surfaceName, StrikeCount = strikes.Length, TenorCount = tenors.Length };
        
        return CreateHandleFor(surfaceObject, surfaceName);
    }, "QL_BuildVolSurface");
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Excel cell → UDF parameter | User-supplied arrays (rates, tenors, strikes, vols) could be malicious or malformed |
| UDF → Handle Store | Handle IDs could be forged, modified, or deleted between calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | UDF input validation | Mitigate | All five UDFs validate array dimensions, null checks, value ranges (strikes/rates non-negative, tenors positive). Invalid input returns clear error message. |
| T-02-02 | Tampering | Handle ID spoofing | Mitigate | Handle IDs are GUID-based, cryptographically random. GetHandleValue fails fast if handle not found (KeyNotFoundException). Attacker cannot guess valid handles. |
| T-02-03 | Information Disclosure | Build details in error messages | Accept | Error messages are generic: "Rates and tenors must have equal length". No sensitive calculation details exposed. |
| T-02-04 | Denial of Service | Large array inputs | Accept | No limits on array size in Phase 1. Assume trusted Excel environment. Spike did not raise memory exhaustion as critical risk. |

## Mitigation Details

**T-02-01 (Input validation):**
- Implementation: Null checks, Length == checks, Range validation
- Example: `if (rates == null || tenors == null || rates.Length != tenors.Length) throw ...`
- Validation: Unit tests (Task 6-7) will test error paths

**T-02-02 (Handle ID spoofing):**
- Implementation: Guid.NewGuid() is cryptographically random; probability of collision vanishes
- Reference: Phase 1 stress test will create 1000+ handles; any collision would be detected

</threat_model>
